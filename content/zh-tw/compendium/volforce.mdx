## Volforce [#volforce]
Volforce 是衡量「實力」的主要方式。當然，這不是百分之百準確，但理論上 Volforce 越高，實力就越強。

你的 Volforce 由最佳 50 次遊玩紀錄組成。每次遊玩後，遊戲會使用以下公式計算該次遊玩的 Volforce。

<Callout>
嚴格來說，這個公式並不正確。Voltex 不會乘以 0.001，而是先無條件捨去至整數，再將該數值加入最佳 50 的總和，最後除以 1000。為了簡化流程，我在公式中加入 0.001；結果相同，只是計算方式不同。
</Callout>
<div className="text-center">
```
(Level*(Score / 10,000,000)*(Grade Coefficient)*(Clear Coefficient)*20)*.001
```

**等級係數／倍率**
</div>

<div className="flex justify-center mt-2">
| S | AAA+ | AAA | AA+ | AA | A+ | A | B | C | D |
|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
|1.05|1.02|1.00|0.97|0.94|0.91|0.88|0.85|0.82|0.80|
</div>

<div className="text-center">
**通關係數／倍率**
</div>

<div className="flex justify-center mt-2">
|PUC|UC|MAXXIVE RATE 通關|EXCESSIVE RATE 通關|EFFECTIVE RATE 通關|Crash|
|:--:|:--:|:--:|:--:|:--:|:--:|
|1.10|1.06\*|1.04|1.02|1.00|0.50|
</div>
\* Konasute 的 UC 為 1.05

例如，在 `18.3 級` 譜面以 `EXCESSIVE RATE` 通關並取得 `9,923,042（S）`，Volforce 的計算方式如下：
<div className="text-center mt-3">
```
(Level*(Score / 10,000,000)*(Grade Coefficient)*(Clear Coefficient)*20)*.001
```
</div>
<div className="mt-3">
```
Level             = 18.3
Score             = 9,923,042
Grade Coefficient = S = 1.05
Clear Coefficient = Excessive Rate = 1.02

  (18.3*(9,923,042 / 10,000,000)*(1.05)*(1.02)*20)*.001
                  = 0.388969354141
```
</div>
結果為 `0.388969354141`。由於 Voltex 會從小數第 3 位開始無條件捨去，因此去除第 3 位之後的數字，答案就是 `.388`。

計算出這個數值後，遊戲會將它與最佳 50 比較。如果這個數值高於最佳 50 中 Volforce 最低的遊玩紀錄，就會獲得兩者的差值。假設最佳 50 中最低的 VF 是 `.370`，遊戲就會以這筆紀錄取代原本的 Volforce，並獲得：
<div className="flex justify-center mt-3">
```
.388 - .370 = 0.018 Volforce
```
</div>

發生這種情況時，增加的數值會顯示在畫面上的 Volforce 旁邊。

要注意的是，計算時只會考慮 **譜面等級、分數、通關類型與等級**。

此外，Volforce 會使用同一張譜面的 **最佳通關類型與最高分數** 計算。例如再次遊玩上一個例子中的譜面，以 `UC` 取得 `9,899,320` 分，Voltex 會將 EXCESSIVE RATE 通關替換為 UC，但保留較高的 `9,923,042` 分。

套用公式後如下：
<div className="mt-3">
```
Level             = 18.3
Score             = 9,923,042 # 保留最高分
Grade Coefficient = S = 1.05
Clear Coefficient = UC = 1.06 # 已變更

  (18.3*(9,923,042 / 10,000,000)*(1.05)*(1.06)*20)*.001
                  = 0.404223054304
```
</div>
結果為 `.404` VF。

注意 `1.02` 已變為 `1.06`，但分數仍使用原本較高的分數。因此將這張譜面的 EXCESSIVE RATE 提升為 UC，即使 UC 分數較低，仍可獲得：
<div className="flex justify-center mt-3">
```
.404 - .388 = .016 Volforce
```
</div>

## 段位 [#skill-ranks]
Volforce 的不同里程碑會對應不同的段位，完整列表如下：

<Image src='/compendium/volforce/volforceranks.png' className="w-100"/>

每個段位都有 4 個獨立的「門檻」。

到達 Imperial 的 `20.00 VF` 後，升段所需的增量會從每 `1.000 VF` 改為每 `4.000 VF`。這是目前 VF 段位的上限；由於可用譜面不足以填滿最佳 50，目前還沒有人能超過 `24.000 VF`。

目前達到 VF 上限的兩位玩家是 [Mini](https://www.youtube.com/@minisdvx7740) 與 [Jakads](https://www.youtube.com/@Jakads_realkk)。他們幾乎每次推出 20 級譜面都能取得 PUC。

## Volforce 潛力資料夾 [#volforce-potential-folder]
首次遊玩 Nabla 後，可能會出現名為「Volforce Potential」的新資料夾。
<Image src="/compendium/volforce/volforcepotential.png"/>

Volforce 潛力資料夾會列出原本有機會進入最佳 50、但目前尚未列入的分數。由於 Nabla 啟動時不會重新計算 Volforce，可能有些譜面具備進入最佳 50 的條件，卻還沒有重新計算。

例如，在 Exceed Gear 時，最佳 50 全部都是 19 級 S 的 EXCESSIVE RATE 通關，當時每筆紀錄價值 `.402` VF。也就是說，必須取得高於 `.402` VF 的分數才能增加 VF。

啟動 Nabla 後，由於加入小數等級，18.6 級 UC 現在價值 `.409` VF，高於 `.402` VF。只要重新遊玩並讓遊戲計算，就有機會進入最佳 50。因此，如果你在 18.6 級歌曲上取得 UC，該紀錄就會顯示在最佳 50 中。

## 注意事項 [#notes]
Volforce 是在機台上計算的。也就是說，如果你在新歌曲上取得提升 Volforce 的紀錄，之後在沒有該歌曲的機台遊玩，Volforce 會按照尚未遊玩該歌曲的狀態計算。

基本上，刷卡登入時，所有分數都會傳送至機台並顯示，機台再根據收到的資料計算 Volforce。

每筆分數的 Volforce 會在取得分數時計算。每完成一張譜面，遊戲就會計算該次遊玩的 Volforce 並與分數一起儲存。因此在 Nabla 中必須重新計算 Volforce，因為 Nabla 不會重新計算每筆分數的 Volforce，只會使用已知資料。

### Nabla [#nabla]
在 Nabla 中，登入時不會計算最高 Volforce。如果將資料從 Exceed Gear 轉移至 Nabla，系統不會重新計算 VF，因此 VF 可能低於應有數值。由於 Nabla 加入小數等級，部分分數的 VF 價值可能高於 Exceed Gear 時的數值；只要重新遊玩一次即可重新計算，取得什麼分數並不重要。

例如，你在 Exceed Gear 的最佳 50 中有一筆 18 級 S；如果 Nabla 將該譜面調整為 18.4 級，只要重新遊玩一次，就能重新計算因等級增加 0.4 而提升的 Volforce。

Nabla 也將 UC 乘數提高了 0.01。
